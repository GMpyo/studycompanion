Which development split should lead the next phase?

A rough map for dividing work across multiple AI coding tools without making everyone edit the same files at once.

Backend-First
Login -> Profile -> Friends -> Ranking
API contracts
Mobile integration

A. Backend first

Build account, cloud save, friend add, and ranking APIs before expanding the game. Best if cross-device and social proof matter most.

Solo Game Loop First
Study -> XP -> Stage clear
Idle rewards and character growth

B. Game first

Make studying feed a solo stage/idle progression system first. Best if the main risk is whether the app feels fun alone.

Parallel Feature Slices
AI 1
Backend
AI 2
Codex UI
AI 3
Game math

C. Parallel slices

Freeze shared interfaces, then split backend, character/dex UI, and solo stage logic into separate branches. Fastest, but needs discipline.

My recommendation

Use C as the coordination method, but sequence the product risk like B -> 2 -> 1: first make the solo game loop fun locally, then polish character/dex presentation, then attach backend social systems once the core loop is worth sharing.
