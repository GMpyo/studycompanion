# 캐릭터 도감 계약

## 목표

캐릭터 도감은 단순 목록이 아니라 수집형 게임의 발견 기록처럼 동작한다. 사용자는 캐릭터를 눌렀을 때 현재 알고 있는 정보, 잠긴 정보, 다음 진화 실루엣을 한 화면에서 확인할 수 있어야 한다.

## 파일 경계

- 도감 전용 데이터: `mobile/src/domain/characterDex.ts`
- 도감 전용 컴포넌트: `mobile/src/components/characterDex/`
- 캐릭터 상세 화면: `mobile/src/app/character/[id].tsx`
- 기존 캐릭터 카탈로그: `mobile/src/domain/catalog.ts`

`catalog.ts`의 8개 캐릭터 ID는 바꾸지 않는다. 깨진 한글 문구를 고치려면 별도 통합 작업에서 처리한다.

## 표시 규칙

해금된 캐릭터:

- 이름
- 짧은 설명
- 성격 설명
- 역할
- 스킬
- 현재까지 해금된 진화 단계 설명
- 아직 도달하지 않은 진화 단계 실루엣

미해금 캐릭터:

- 이름 대신 `???`
- 구체 설명 숨김
- 해금 힌트만 표시
- 모든 진화 단계는 잠김 상태로 표시

## 통합 요청사항

도감 목록 화면에서 상세 화면으로 이동하는 연결은 통합 단계에서 처리한다.

권장 경로:

```text
/character/[id]
```

도감 목록에서 해금 캐릭터와 미해금 캐릭터 모두 상세 화면을 열 수 있게 하면, 미해금 캐릭터의 해금 힌트도 자연스럽게 보여줄 수 있다.
