# 의대생 공부 동반 캐릭터 앱 MVP 설계

## 목적

의과대학생이 공부 시간을 기록하면서 귀여운 캐릭터를 키우고, 부담 없이 다시 공부를 시작하고 싶어지는 개인용 앱을 만든다. 첫 버전의 목표는 친구 기능이나 경쟁을 구현하는 것이 아니라, `공부 완료 -> 캐릭터 성장 -> 애착 형성 -> 다음 공부 시작` 루프가 실제로 즐거운지 검증하는 것이다.

앱은 공부 기록과 캐릭터 육성을 동등한 비중으로 보여준다. 타이머가 중심이고 캐릭터가 장식인 앱도 아니며, 게임을 하기 위한 과제처럼 공부를 강요하는 앱도 아니다.

## 사용자와 제품 원칙

- 초기 핵심 사용자는 iPhone을 사용하는 의대생 개인 사용자다.
- 공부를 많이 하지 못한 날이나 쉬었던 기간을 벌주지 않는다.
- 장시간 공부 경쟁이나 순위 압박을 만들지 않는다.
- 짧은 집중 세션도 의미 있게 보상하여 다시 시작하기 쉽게 한다.
- 향후 친구 협동 레이드로 확장할 수 있지만, MVP는 혼자 육성하는 경험에 집중한다.

## MVP 핵심 경험

사용자는 홈에서 대표 캐릭터를 만나고 곧바로 집중 공부를 시작한다. 공부가 끝나면 시간 기록과 함께 성장 보상을 받고, 캐릭터에게 간식을 주거나 성장 및 부화 연출을 확인한다. 도감에서 모은 캐릭터와 앞으로 발견할 캐릭터를 보며 다음 세션을 시작할 이유를 얻는다.

핵심 흐름:

1. 홈에서 캐릭터와 오늘의 공부 상태 확인
2. 집중 타이머 시작 및 세션 완료
3. 완료한 시간에 따라 경험치, 간식 재화, 발견 포인트 수령
4. 대표 캐릭터 성장, 간식 주기, 친밀도 확인
5. 알 부화, 성장 단계 변화, 도감 해금 확인
6. 다음 공부 세션 시작

## MVP 범위

### 포함 기능

- 집중 타이머와 완료된 공부 세션 기록
- 대표 캐릭터가 생활하는 홈 화면
- 공부 완료 보상: 성장 경험치, 간식 재화, 발견 포인트
- 첫 알 선택 또는 지급, 부화와 성장 단계 변화
- 간식 주기와 친밀도 상승
- 보유 및 미발견 캐릭터를 보여주는 도감
- 캐릭터별 향후 레이드용 역할과 대표 스킬 표시
- 쉬었다 돌아온 사용자를 환영하는 복귀 응원선물
- 오늘 및 이번 주 중심의 간결한 공부 기록 확인

### 제외 기능

- 친구 초대, 파티, 채팅
- 협동 보스와 자동 전투
- 경쟁 순위표
- 장비, 복잡한 스탯, 스킬 강화 트리
- 과목별 심층 통계, 시험 일정, 의학 학습 콘텐츠 연동

## 성장 및 보상 구조

### 보상 재화

- `성장 경험치`: 현재 대표 캐릭터의 부화 및 성장 조건에 사용한다.
- `간식 재화`: 선택한 캐릭터에게 주어 친밀도를 높인다.
- `발견 포인트`: 모아서 새로운 알 또는 도감 발견 기회를 연다.

### 성장 단계

캐릭터는 `알 -> 아기 -> 성장기 -> 성체` 네 단계로 성장한다. 성장에는 경험치와 친밀도 조건을 함께 적용한다. 사용자가 단순히 타이머를 오래 켜 두기보다, 함께 키우는 감각을 가지도록 하기 위함이다.

### 보상 안전장치

- 짧은 세션도 의미가 있도록 최소 15분 완료부터 기본 보상을 지급한다.
- 긴 세션은 더 많은 보상을 주되, 지나친 장시간 공부를 유도하지 않도록 일일 보상 효율은 완만하게 감소시킨다.
- 쉬는 동안 캐릭터의 경험치, 친밀도, 성장 상태, 도감 진행은 감소하지 않는다.
- 연속 출석 보상을 핵심 성장 조건으로 만들지 않는다.
- 같은 완료 세션에서 보상이 중복 지급되지 않게 세션별 지급 상태를 저장한다.

### 복귀 응원선물

하루 이상 완료된 공부 세션이 없었다가 사용자가 다시 첫 세션을 완료하면 `복귀 응원선물`을 지급한다.

- 선물 종류: 간식, 작은 방 꾸미기 아이템, 친밀도 보너스 중 하나
- 연출: 대표 캐릭터의 짧은 환영 메시지와 선물 전달 애니메이션
- 휴식 기간이 길수록 보상이 계속 커지지는 않는다. 쉬는 기간을 보상 획득 수단으로 만들지 않는다.
- 선물의 목적은 손실 복구가 아니라 다시 시작한 행동을 따뜻하게 환영하는 것이다.

## 캐릭터 콘텐츠 범위

첫 버전은 적은 수의 캐릭터를 정성스럽게 보여주는 데 집중한다.

- 시작 알: 3종 중 하나를 선택
- 추가 발견 캐릭터: 5종
- 성장 단계: 캐릭터별 4단계
- 역할: 공격, 방어, 회복, 지원
- 대표 스킬: 캐릭터별 1개

MVP에서 역할과 대표 스킬은 수집 동기를 주는 설명 및 연출 요소다. 실제 전투 계산에는 아직 사용하지 않으며, 추후 자동 협동 레이드 구현 시 기존 캐릭터 데이터를 그대로 활용한다.

## 주요 화면

### 홈

대표 캐릭터가 작은 방이나 생활 공간에서 움직이며 사용자를 맞이한다. 오늘 공부시간, 집중 시작 버튼, 다음 성장까지의 진행도, 간식 주기, 복귀 응원선물 연출을 이 화면에 배치한다. 공부와 캐릭터 경험을 동등하게 느끼게 하는 핵심 화면이다.

### 집중 타이머

집중 중에는 캐릭터가 조용히 함께 공부하거나 간단히 응원한다. 집중을 방해하지 않도록 화려한 이벤트와 육성 조작은 세션 완료 뒤에만 제공한다.

### 세션 완료 보상

완료한 시간, 획득 경험치, 간식 재화, 발견 포인트를 보여준다. 부화나 성장 조건을 만족하면 바로 축하 연출로 이어진다. 복귀 세션인 경우 응원선물도 이 흐름에서 전달한다.

### 캐릭터 및 도감

보유 캐릭터, 미발견 실루엣, 성장 단계, 친밀도, 역할, 대표 스킬을 확인한다. 미래의 레이드 기능이 추가되더라도 이 정보 구조는 유지한다.

### 기록

오늘과 이번 주 공부시간, 최근 완료 세션을 간결하게 제공한다. MVP에서는 의대 전용 세분화 기능보다 육성 루프의 반복 사용 여부를 검증하는 데 집중한다.

## 데이터 흐름과 예외 처리

세션 완료 시 처리 순서:

1. 타이머에서 유효한 완료 시간을 확정한다.
2. 완료 세션을 고유 식별자와 함께 저장한다.
3. 시간 및 일일 효율 규칙에 따라 보상을 계산한다.
4. 대표 캐릭터에 경험치를 적용하고 지급 재화를 저장한다.
5. 경험치와 친밀도 조건을 확인하여 성장 이벤트를 개방한다.
6. 복귀 조건을 만족하면 응원선물을 한 번 지급한다.
7. 홈, 도감, 기록 화면에 변경 상태를 반영한다.

예외 규칙:

- 타이머를 중단하면 실제 완료한 시간만 기록하며, 15분 미만은 기본 보상 대상에서 제외한다.
- 앱이 닫혔다 다시 열려도 진행 중 세션 시작 시간을 바탕으로 복원할 수 있게 한다.
- 날짜가 바뀌어도 완료 기록과 캐릭터 상태가 손실되지 않는다.
- 세션 완료 처리 도중 재시도가 발생해도 같은 보상이 두 번 지급되지 않게 한다.

## 기술 방향

### 권장 스택

`Expo + React Native + Expo Router` 기반의 단일 프로젝트로 시작한다.

- 초기 MVP는 웹으로 실행하고 배포하여 빠르게 사용성을 확인한다.
- iPhone 사용자는 Safari에서 웹 버전을 홈 화면에 추가해 앱에 가까운 방식으로 시험할 수 있다.
- 같은 프로젝트 기반에서 이후 iOS 앱 빌드 및 TestFlight/App Store 배포로 확장한다.
- 개발 환경이 Windows여도 Expo EAS Build의 클라우드 macOS 빌드 환경을 사용하여 iOS 빌드를 진행할 수 있다.

### 웹 우선 검증 순서

1. Expo 웹으로 개인 육성 MVP를 개발한다.
2. iPhone Safari 및 홈 화면 추가 환경에서 직접 사용한다.
3. 타이머 안정성, 캐릭터 애착, 복귀 경험이 유효한지 확인한다.
4. 충분히 반복 사용된다면 iOS 빌드와 TestFlight 배포를 진행한다.
5. 육성 루프가 검증된 뒤 친구 파티 및 자동 협동 레이드를 설계한다.

### 배포 관련 전제

- 웹 MVP는 App Store 심사 없이 테스트할 수 있다.
- 정식 iOS App Store 배포를 진행할 시 Apple Developer Program 가입이 필요하며, 2026년 5월 확인 기준 공식 연회비는 99 USD이다.
- 네이티브 iOS 개발 도구인 Xcode는 macOS에서 제공되지만, Expo EAS Build는 iOS 바이너리를 Expo의 클라우드 macOS 빌드 환경에서 생성한다.

## 미래 확장: 친구 자동 협동 레이드

MVP 이후에는 사용자가 키운 캐릭터를 친구들과 함께 사용하는 비동기 자동 레이드를 추가한다.

- 친구 3~5명이 파티를 만들고 각자 대표 캐릭터를 등록한다.
- 캐릭터의 역할과 대표 스킬 조합이 자동 전투 결과에 반영된다.
- 공부시간은 전투력 경쟁으로 직접 환산하기보다 성장 및 준비 자원을 제공한다.
- 모든 참여자는 기본 보상을 받고, 파티 승리 시 추가 보상을 받는다.
- 순위 경쟁보다 함께 보스를 물리치고 수집 보상을 얻는 경험을 우선한다.

보스의 테마는 의학이나 질병으로 제한하지 않으며, 캐릭터 세계관에 맞는 자유로운 판타지 테마를 사용한다.

## 성공 기준

MVP는 다음 행동이 나타날 때 성공 가능성이 있다고 판단한다.

- 사용자가 첫 알을 부화시키기 위해 공부 세션을 반복 완료한다.
- 사용자가 캐릭터를 성체까지 키우고 새 캐릭터 발견을 시도한다.
- 며칠 쉬었던 사용자가 복귀 응원선물을 경험한 뒤 다시 세션을 이어간다.
- 기록 화면과 캐릭터 화면이 모두 반복 사용되어 공부 또는 육성 중 하나가 장식 기능으로 남지 않는다.

## 검증 항목

- 세션 완료 시간에 따른 보상이 정확히 한 번 지급되는지 확인한다.
- 15분 미만 세션과 보상 효율 감소 규칙이 의도대로 동작하는지 확인한다.
- 앱 종료 및 재진입 후 타이머와 미지급 보상 상태가 복원되는지 확인한다.
- 성장 조건과 복귀 응원선물 지급 조건이 중복 없이 동작하는지 확인한다.
- iPhone Safari와 홈 화면 추가 환경에서 타이머, 화면 전환, 저장 상태 사용성을 직접 점검한다.
